Programación orientada a objetos

Clases y objetos.

Programación orientada a objetos (POO) es un paradigma de programación que usa objetos y sus interacciones para la construcción de programas de computador. Para esto, la POO usa encapsulación, abstracción, herencia y polimorfismo. La POO es el sucesor de la programación procedimental (estructural), la cual describe programas como unidades de codigo reutilizable (procedimientos) que definen o hacen uso de parámetros de entrada y salida. Los programas procedimentales consisten de procedimientos que se invocan unos a otros.
Características.
  • Encapsulación. Se utiliza para ocultar detalles innecesarios en nuestras clases suministrando una interface clara y simple para trabajar con ellos.
  • Herencia. Sirve para la reutilización del código mejorando con esto la funcionalidad.
  • Abstracción. Recoge las características principales de los objetos e ignorando otros detalles.
  • Polimorfismo. Trata de las diferentes funcionalidades que se le pueden dar a un mismo método por medio de la variación de sus declaraciones.
Objeto. Podría decirse que es la representación simplificaca de una clase y que es utilizado para acceder a los métodos definidos en ésta.
Clase. Define el conjunto de médodos, procedimientos y parámetros por medio de los cuales se hace la abstracción y manejo de la información de una entidad.
Instancia. Es una representación de una clase dentro de otra clase. Una clase puede ser instanciada (representada) en una o más clases.

Ejemplo.

Aquí se tienen dos clases, una es la clase principal program y otra es la clase VSehiculo. La clase Vehiculo tiene cinco variables miembro: marca, modelo, cilindraje, tipo y precio.
Con cada variable miembro se encuentra asociado un par de métodos get y set. El método set permite la actualización de la variable a él asociada y el método get permite la lectura de tal variable. El acceso a cada método se hace por medio del constructor de la clase el cual se encuentra sobrecargado: un constructor con parámetros y un constructor vacío. En la clae program se realizan dos instanciados de la clase vehiculo. Uno sirve para accesar cada uno de los miembros de esta clase y el otro es utilizado para accesar los métodos que se agreguen a la clase vehiculo, en este caso, el método actualizar.

¿Cómo se realiza el instanciado?
De la clase Vehiculo se crea un objeto llamado vehiculo. Este objeto realmente podría tener cualquier nombre similar: vehiculo, vehi, ve, etc.
Es a través de este objeto como se accede a cada uno de los métodos de la clase Vehiculo. Por ejemplo, para acceder al método Marca se escribirá vehiculo.Marca y así para los demás métodos, vehiculo.Modelo, vehiculo.Tipo, vehiculo.Precio y vehiculo.cilindraje.

Cuando se hace una lectura como en el caso: Console.WriteLine("\nMarca: " + vehiculo.Marca); lo que se está haciendo es invocar al get del método Marca el cual retorna el valor actual de marca que previamente ha sido actualizado por medio del constructor con parámetros mediante el par de instrucciones:
String marc = Console.ReadLine(); y
Vehiculo vehiculo = new Vehiculo(marc, mod, cil, tip, pre);
Lo mismo se da para los demás métodos de la clase Vehiculo.
Otra instancia de la clase Vehiculo se da como: Vehiculo veh = new Vehiculo();
Obsérvese que se hace uso de la misma clase pero se crea un objeto diferente: veh y se invoca al constructor vacío (el que no tiene parámetros).
Es a través de este objeto como se accede al método actualizar.

Programa de ejemplo.

            namespace Clases_Uno
            {


            internal class Vehiculo
            {
            //Miembros de la clase
            String marca;
            int modelo;
            int cilindraje;
            String tipo;
            Double precio;

            //Constructur de la clase vehiculo con parámetros


            //Constructur de la clase vehiculo sin parámetros
            public Vehiculo()
            {
            }

            public Vehiculo(string marca, int modelo, int cilindraje, string tipo, double precio)
            {
            this.marca = marca;
            this.modelo = modelo;
            this.cilindraje = cilindraje;
            this.tipo = tipo;
            this.precio = precio;
            }

            public string Marca // property
            {
            get { return marca; } // get method
            set { marca = value; } // set method
            }

            public int Modelo // property
            {
            get { return modelo; } // get method
            set { modelo = value; } // set method
            }

            }
            }

            Class program.


            //Instanciar la clase vehiculo

            Console.WriteLine("Cuál es la marca del vehículo");
            String marc = Console.ReadLine();
            Console.WriteLine("Cuál es el modelo del vehículo");
            int mod = int.Parse(Console.ReadLine());
            Console.WriteLine("Cuál es el cilindraje del vehículo");
            int cil = int.Parse(Console.ReadLine());
            Console.WriteLine("Cuál es el tipo de vehículo");
            String tip = Console.ReadLine();
            Console.WriteLine("Cuál es el precio del vehículo");
            double pre = double.Parse(Console.ReadLine());

            Vehiculo vehiculo = new Vehiculo(marc, mod,
            cil, tip, pre);

            Console.WriteLine("\nMarca: " + vehiculo.Marca);
            Console.WriteLine("Modelo: " + vehiculo.Modelo);
            Console.WriteLine("Cilindraje: " + vehiculo.Cilindraje);
            Console.WriteLine("Tipo: " + vehiculo.Tipo);
            Console.WriteLine("Precio: " + vehiculo.Precio);
